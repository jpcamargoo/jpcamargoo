 Como penso em sistemas
- Arquitetura-first: antes de escrever código, defino limites, contratos e fluxos.  
- IPO: cada módulo precisa ter entradas claras, processamento explícito e saídas rastreáveis.  
- Observabilidade: logs estruturados, métricas e rastreamento desde o início.  
- Normalização intencional: dados limpos, consistentes e auditáveis.  
- Documentação como parte do produto: não como anexo.  
- Automação sempre que possível: pipelines, geração de relatórios, ETL, testes.
